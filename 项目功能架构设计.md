# 综测管理系统 — 全栈项目功能架构设计文档

> 基于原 Python 桌面程序，重构为本地部署的 B/S 架构 Web 应用  
> UI 风格参照 mizu-light-astro-theme（Tailwind CSS + oklch 配色 + Clash Display/Grotesk 字体）  
> 最后更新：2026-02-24

---

## 一、技术栈选型

| 层 | 技术 | 说明 |
|----|------|------|
| **前端** | Astro 5 + React 19 + Tailwind CSS 4 | SSR/Islands 架构，复用 mizu-light-astro-theme 设计语言 |
| **后端** | Node.js + Express（或 Fastify） | RESTful API + WebSocket（实时保存） |
| **数据库** | SQLite（默认）/ PostgreSQL（可选） | 本地部署优先 SQLite 零配置；支持后续迁移到 PostgreSQL |
| **ORM** | Prisma | 类型安全的数据库访问层，迁移方便 |
| **认证** | JWT + bcrypt | 无需第三方服务，本地可用 |
| **实时通信** | WebSocket (ws / Socket.IO) | 分数编辑实时保存 & 多端同步 |
| **文件处理** | ExcelJS (xlsx 读写) | 替代原有 openpyxl，功能等价 |
| **打包/部署** | Docker Compose / PM2 | 一键启动前后端 + 数据库 |

## 三、数据库设计

### 3.1 ER 关系图

```
User (用户表)
  │ 1
  │
  ├── manages ──► Grade/Class (通过 class_id 关联)
  │
Student (学生表)
  │ 1                     ▲
  │                       │ N
  ├──── has ────► Score (分数表)
  │                       │
  │                       ├── score_type (维度)
  │                       └── value / remark
  │
  └──── belongs_to ──► Class (班级表)
                          │
                          └── belongs_to ──► Grade (年级表)
```

### 3.2 表结构

#### users — 用户表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO_INCREMENT | 主键 |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt 加密后的密码 |
| role | ENUM('admin', 'monitor') | NOT NULL | 角色：管理员 / 班长 |
| class_id | INTEGER | FK → classes.id, NULLABLE | 班长关联的班级（管理员为 NULL） |
| display_name | VARCHAR(50) | | 显示名称（如 "计科1班班长"） |
| created_at | DATETIME | DEFAULT NOW | 创建时间 |
| updated_at | DATETIME | DEFAULT NOW | 更新时间 |

#### grades — 年级表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO_INCREMENT | 主键 |
| name | VARCHAR(20) | UNIQUE, NOT NULL | 年级名称，如 "2023级" |
| created_at | DATETIME | DEFAULT NOW | 创建时间 |

#### classes — 班级表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO_INCREMENT | 主键 |
| grade_id | INTEGER | FK → grades.id, NOT NULL | 所属年级 |
| name | VARCHAR(50) | NOT NULL | 班级名称，如 "计算机科学与技术1班" |
| created_at | DATETIME | DEFAULT NOW | 创建时间 |
| **UNIQUE** | | (grade_id, name) | 同年级下班级名称唯一 |

#### students — 学生表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO_INCREMENT | 主键 |
| student_no | VARCHAR(20) | UNIQUE, NOT NULL | 学号 |
| name | VARCHAR(50) | NOT NULL | 姓名 |
| class_id | INTEGER | FK → classes.id, NOT NULL | 所属班级 |
| created_at | DATETIME | DEFAULT NOW | 创建时间 |
| updated_at | DATETIME | DEFAULT NOW | 更新时间 |

#### academic_years — 测评学年表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO_INCREMENT | 主键 |
| name | VARCHAR(30) | UNIQUE, NOT NULL | 学年名称，如 "2025-2026学年" |
| is_current | BOOLEAN | DEFAULT FALSE | 是否为当前激活学年 |
| created_at | DATETIME | DEFAULT NOW | 创建时间 |

#### scores — 分数表（核心表，逐维度存储）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO_INCREMENT | 主键 |
| student_id | INTEGER | FK → students.id, NOT NULL | 学生 |
| academic_year_id | INTEGER | FK → academic_years.id, NOT NULL | 学年 |
| category | VARCHAR(30) | NOT NULL | 分数维度（见下方枚举） |
| value | DECIMAL(6,2) | DEFAULT 0 | 分数值 |
| remark | TEXT | NULLABLE | 备注（来自个人综测填写表） |
| updated_by | INTEGER | FK → users.id | 最后修改人 |
| updated_at | DATETIME | DEFAULT NOW | 最后修改时间 |
| **UNIQUE** | | (student_id, academic_year_id, category) | 同学生同学年同维度唯一 |

**category 枚举值：**

| category 值 | 中文名 | 分数上限 | 备注 |
|-------------|--------|---------|------|
| `moral` | 德育测评 | 100 | |
| `academic` | 学业学术素质 | 60 | 通常批量导入 |
| `innovation` | 创新与实践能力 | 13 | |
| `sports_base` | 体育基础分 | — | 由导入写入，不可手动修改 |
| `sports_reward` | 体育奖励分 | 3 | 手动填写 0~3 |
| `sports_total` | 体育总分 | 7 | = sports_base + sports_reward，自动计算 |
| `aesthetics` | 美育 | 6 | |
| `labor` | 劳动教育 | 4 | |
| `public_service` | 公益服务与社会工作 | 10 | |
| `bonus` | 附加分 | 5 | |
| `total` | 总分 | — | 所有维度之和，自动计算 |

#### import_logs — 导入日志表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | INTEGER | PK, AUTO_INCREMENT | 主键 |
| type | VARCHAR(20) | NOT NULL | 导入类型：academic / sports / personal_form |
| filename | VARCHAR(255) | | 原始文件名 |
| success_count | INTEGER | DEFAULT 0 | 成功条数 |
| fail_count | INTEGER | DEFAULT 0 | 失败条数 |
| fail_details | JSON | NULLABLE | 失败记录详情 |
| imported_by | INTEGER | FK → users.id | 操作人 |
| created_at | DATETIME | DEFAULT NOW | 导入时间 |

## 四、用户角色与权限体系

### 4.1 角色定义

| 角色 | 标识 | 说明 |
|------|------|------|
| **管理员** | `admin` | 拥有系统全部权限，可管理所有年级、班级、学生、分数 |
| **班长** | `monitor` | 绑定到某一班级，仅可查看和操作本班数据 |

### 4.2 权限矩阵

| 功能 | 管理员 | 班长 |
|------|--------|------|
| 查看所有年级/班级/学生 | ✅ | ❌（仅本班） |
| 编辑任意学生分数 | ✅ | ❌（仅本班） |
| 分数实时保存 | ✅ | ✅ |
| 添加新学生 | ✅ | ❌ |
| 删除学生（单个） | ✅ | ❌ |
| 删除学生（整个年级） | ✅ | ❌ |
| 批量导入学业学术素质成绩 | ✅ | ❌ |
| 批量导入体育基础分 | ✅ | ❌ |
| 批量导入个人综测填写表 | ✅ | ✅（仅本班） |
| 批量生成班长账号并导出 | ✅ | ❌ |
| 导出附件2（综测成绩汇总表） | ✅（所有班级） | ❌ |
| 导出附件4（学年总评表） | ✅（所有班级） | ❌ |
| 管理学年 | ✅ | ❌ |
| 查看导入日志 | ✅ | ✅（仅本班相关） |

### 4.3 账号管理流程

#### 管理员账号
- 系统首次部署时自动生成一个默认管理员账号（`admin` / 初始密码）
- 管理员可在设置中修改自己的密码
- 支持创建多个管理员账号

#### 班长账号批量生成
1. 管理员选择目标年级（或全部年级）
2. 系统自动为每个班级生成一个班长账号：
   - **用户名规则**：`monitor_{年级编号}_{班级序号}`（如 `monitor_2023_jk1`）
   - **初始密码规则**：随机 8 位字母数字组合
   - **显示名称**：`{年级}{班级名称}班长`
3. 已有账号的班级可选择 **跳过** 或 **重新生成**（重置密码）
4. 生成完毕后导出为 Excel 文件，包含列：年级、班级、用户名、初始密码
5. 导出文件供下发给各班班长

#### 5.3.2 仪表盘页面 (`/dashboard`)

#### 5.3.3 分数编辑页面 (`/scores/:classId`) — **核心页面**

```
┌────────────────────────────────────────────────────────┐
│ Header: 导航栏 + 角色标识 + 退出登录                    │
├────────────────────────────────────────────────────────┤
│ Breadcrumb: 首页 > 分数管理 > 2023级 > 计科1班         │
├────────────────────────────────────────────────────────┤
│                                                        │
│  [年级选择 ▼] [班级选择 ▼] [搜索学生: ________]        │
│                                                        │
│  ┌──────────────────────────────────────────────────┐  │
│  │ 学号    │ 姓名 │ 德育 │ 学业 │ 创新 │ 体育 │ ... │  │
│  ├─────────┼──────┼──────┼──────┼──────┼──────┼─────┤  │
│  │ 2325... │ 张三 │ [85] │ 36.0 │ [10] │ 5.58 │ ... │  │
│  │ 2325... │ 李四 │ [90] │ 42.0 │ [ 8] │ 4.20 │ ... │  │
│  │         │      │      │      │      │      │     │  │
│  │  (可编辑的单元格实时保存，失焦/回车即触发保存)      │  │
│  └──────────────────────────────────────────────────┘  │
│                                                        │
│  状态栏: ✅ 已自动保存 15:30:42  │ 完成度: 35/42       │

│  [计算总分并排序]  [导入个人综测表]                      │
└────────────────────────────────────────────────────────┘
```

**核心特性—实时保存：**
- 每个可编辑单元格采用 **Inline Edit** 模式（点击即变为输入框）
- **触发保存时机**：
  - 输入框 `onBlur`（失焦）时自动保存
  - 按 Enter 键保存并跳到下一行同列
  - 按 Tab 键保存并跳到同行下一列
- **保存流程**：前端通过 WebSocket 发送 `{ studentId, category, value, remark }` → 后端接收并写库 → 返回确认 → 前端更新状态栏 "✅ 已自动保存 HH:MM:SS"
- **Debounce 策略**：输入过程中每次按键后 300ms 内无新输入则自动保存（避免频繁请求）
- **乐观更新**：前端先显示新值，保存失败时回滚并标红提示
- **离线容错**：本地 localStorage 暂存未保存的修改，网络恢复后自动重试

**备注功能：**
- 每个分数单元格右侧有一个小图标 💬，悬浮显示备注内容
- 点击图标弹出 Popover 输入框，可编辑备注
- 备注同样实时保存

**体育分数联动：**
- 体育基础分单元格为只读（灰色背景）
- 体育奖励分可编辑，修改后自动计算体育总分并更新显示
- 总分列始终为自动计算，不可编辑

#### 5.3.4 学生管理页面 (`/students`) — 管理员专属

**功能区块：**

1. **学生列表**
   - 支持按年级/班级筛选
   - 表格展示：学号、姓名、年级、班级、操作（编辑/删除）
   - 支持分页 + 搜索

2. **添加学生**
   - 单个添加：表单输入学号、姓名、选择年级+班级
   - 批量添加：上传 Excel 文件，格式与原 `student_data.xlsx` 一致
     - 列：学号、姓名、年级、班级
     - 自动创建不存在的年级和班级
   - **格式提醒**：页面上展示 student_data 的格式说明和示例模板下载

3. **删除学生**
   - **单个删除**：点击学生行的删除按钮 → 二次确认弹窗 → 确认后删除学生及其所有分数数据
   - **整个年级删除**：选择年级 → 显示该年级学生总数 → 输入年级名称确认（防误操作） → 删除该年级所有学生、班级、分数数据
   - 删除操作记入操作日志

4. **编辑学生信息**
   - 可修改学号、姓名、所属班级（转班）
   - 转班时分数数据随学生迁移

#### 5.3.5 批量导入页面 (`/import`) — 管理员专属

**三种导入类型：**

| 导入类型 | 文件格式 | 计算公式 | 匹配方式 |
|---------|---------|---------|---------|
| 学业学术素质 | A列学号 B列姓名 F列绩点 | `(绩点+2.5)×8` 四舍五入两位 | 按学号全局匹配 |
| 体育基础分 | A列学号 B列姓名 H列基础分 | `基础分×0.04` 四舍五入两位 | 按学号全局匹配 |
| 个人综测填写表 | 见 5.3.6 | 直接取值 | 按学号匹配本班 |

**导入流程 UI：**
1. 选择导入类型 → 上传文件 → 显示文件预览（前 10 行）
2. 确认导入 → 后端处理 → 实时显示进度条
3. 完成后显示结果卡片：成功 N 条、失败 M 条
4. 失败记录展示为表格，支持一键导出为 Excel

#### 5.3.6 个人综测填写表导入 (`/import/personal`) — 班长可访问

- 班长上传个人综测填写表 Excel 文件（可一次上传多个学生的文件或一份汇总表）
- **文件格式识别**：按照学校发放的个人综测填写表的格式自动解析
- **导入内容**：
  - 各维度分数
  - 表格内的备注内容（对应 scores 表的 remark 字段）
- **导入规则**：
  - 上传时自动按学号匹配到当前班级的学生
  - 如学号不在本班范围内 → 记录为失败
  - 已有分数数据 → 覆盖更新（保留最新版本）
- **操作约束**：班长只能为本班学生导入

## 七、核心业务逻辑

### 7.1 分数实时保存流程

```
用户编辑单元格
       │
       ▼
  Debounce 300ms
       │
       ▼
  输入校验（前端）
  ├── 是否为有效数字？
  ├── 是否超过分数上限？
  └── 体育奖励分是否在 0~3？
       │
       ├── 校验失败 → 单元格标红 + Tooltip 提示
       │
       ▼ 校验通过
  乐观更新 UI（立即显示新值）
       │
       ▼
  WebSocket 发送 score:update
       │
       ▼
  后端接收 → 权限校验 → 业务校验 → 写入数据库
       │
       ├── 成功 → 返回 score:updated
       │         → 广播 score:sync 给同班其他在线用户
       │         → 如果是 sports_reward 变更，自动重算 sports_total
       │         → 重算 total
       │         → 前端状态栏更新 "✅ 已保存"
       │
       └── 失败 → 返回 score:error
                → 前端回滚值 + 显示错误
                → 存入 localStorage 待重试
```

### 7.2 体育分数联动计算

```
导入体育基础分:
  raw_base = Excel H列值
  sports_base = round(raw_base × 0.04, 2)
  sports_total = sports_base + sports_reward（如已存在）
  → 写入 sports_base, sports_total

编辑体育奖励分:
  sports_reward = 用户输入值（0~3）
  sports_total = sports_base（如已存在）+ sports_reward
  → 写入 sports_reward, sports_total

总分计算:
  total = moral + academic + innovation + sports_total
        + aesthetics + labor + public_service + bonus
```

### 7.3 个人综测填写表导入解析

```
解析流程:
  1. 读取上传的 Excel 文件
  2. 识别学号（定位学号所在单元格）
  3. 逐项提取分数及备注:
     - 德育测评分数 + 备注
     - 创新与实践能力分数 + 备注
     - 体育奖励分 + 备注
     - 美育分数 + 备注
     - 劳动教育分数 + 备注
     - 公益服务与社会工作分数 + 备注
     - 附加分 + 备注
  4. 按学号匹配学生
  5. 写入 scores 表（value + remark 字段）
  6. 自动重算 sports_total 和 total
```

### 7.4 分数校验规则

| 维度 | 校验规则 |
|------|---------|
| 德育测评 | 0 ≤ value ≤ 100，必须为数字 |
| 学业学术素质 | 0 ≤ value ≤ 60，必须为数字 |
| 创新与实践能力 | 0 ≤ value ≤ 13，必须为数字 |
| 体育基础分 | 必须为数字（上限由导入数据决定） |
| 体育奖励分 | 0 ≤ value ≤ 3，必须为数字 |
| 体育总分 | 0 ≤ value ≤ 7，自动计算 |
| 美育 | 0 ≤ value ≤ 6，必须为数字 |
| 劳动教育 | 0 ≤ value ≤ 4，必须为数字 |
| 公益服务与社会工作 | 0 ≤ value ≤ 10，必须为数字 |
| 附加分 | 0 ≤ value ≤ 5，必须为数字 |

## 十、安全设计

| 安全措施 | 实现方式 |
|---------|---------|
| 密码加密 | bcrypt (salt rounds = 12) |
| 接口鉴权 | JWT Token，有效期 24h，支持 Refresh Token |
| 权限控制 | 中间件层统一校验角色 + 数据归属 |
| 输入校验 | 前后端双重校验（Zod schema） |
| SQL 注入防护 | Prisma ORM 参数化查询 |
| XSS 防护 | Astro 默认转义 + CSP Header |
| CSRF 防护 | SameSite Cookie + CSRF Token |
| 操作日志 | 关键操作（删除、导入）记录审计日志 |
| 数据备份 | SQLite 定时自动备份（每日），保留最近 7 天 |
