## 快速开始

### 环境要求

- **Node.js** ≥ 18
- **npm** ≥ 9

### 安装与启动

```bash
# 1. 进入项目目录
cd packages

# 2. 安装根目录依赖（concurrently）
npm install

# 3. 安装后端依赖
cd backend && npm install

# 4. 初始化数据库 & 创建默认管理员账号
npx prisma db push
npx prisma db seed

# 5. 安装前端依赖
cd ../frontend && npm install

# 6. 返回根目录，一键启动前后端
cd ..
npm run dev
```

启动后访问：

| 服务 | 地址 |
|------|------|
| **前端** | http://localhost:3000 |
| **后端 API** | http://localhost:4000 |

### 默认管理员账号

| 用户名 | 密码 |
|--------|------|
| `admin` | `admin123` |

> ⚠️ 首次登录后请立即在「系统设置」中修改默认密码

### 常用命令

```bash
# 同时启动前后端开发服务器
npm run dev

# 仅启动后端
npm run dev:backend

# 仅启动前端
npm run dev:frontend

# 构建前端生产版本
npm run build

# 重置数据库（清除所有数据并重建）
npm run db:reset
```

## 系统架构

```
浏览器 (Astro + React + Tailwind CSS)
    │
    ├── HTTP REST API ──► Express 后端 (Port 4000)
    │                         │
    └── WebSocket ──────► ws 实时通信
                              │
                         Prisma ORM
                              │
                         SQLite 数据库
```

| 层 | 技术 |
|----|------|
| 前端框架 | Astro 5 + React 19 |
| UI 样式 | Tailwind CSS 4 (oklch 暖橙配色) |
| 后端框架 | Express 4 + TypeScript |
| 数据库 | SQLite (Prisma ORM) |
| 认证 | JWT + bcrypt |
| 实时通信 | WebSocket (ws) |
| 文件处理 | ExcelJS (xlsx 读写) |

## 角色与权限

系统分为两种角色：

### 管理员 (admin)

拥有系统全部权限，包括：

- 管理所有年级、班级、学生
- 编辑任意学生的综测分数
- 批量导入学业成绩、体育基础分、个人综测表
- 导出附件2/附件4/全量ZIP
- 批量生成和管理班长账号
- 管理学年、修改密码

### 班长 (monitor)

绑定到某一班级，权限范围限于本班：

- 查看和编辑本班学生分数
- 导入本班个人综测填写表
- 查看本班导入日志
- 修改自己的密码

### 权限矩阵

| 功能 | 管理员 | 班长 |
|------|:------:|:----:|
| 查看所有年级/班级/学生 | ✅ | ❌（仅本班） |
| 编辑学生分数 | ✅ 全部 | ✅ 仅本班 |
| 添加/删除学生 | ✅ | ❌ |
| 导入学业成绩 | ✅ | ❌ |
| 导入体育基础分 | ✅ | ❌ |
| 导入个人综测表 | ✅ | ✅ 仅本班 |
| 导出附件 | ✅ | ❌ |
| 管理账号 | ✅ | ❌ |
| 管理学年 | ✅ | ❌ |
| 修改自己密码 | ✅ | ✅ |

  - **学生管理** → 跳转到学生管理

  - **数据导出** → 跳转到导出页面

### 3. 分数编辑

**入口**：http://localhost:3000/scores

这是系统的**核心功能页面**，用于查看和编辑学生的综测分数。

#### 操作步骤

1. **选择年级** — 从下拉菜单选择目标年级（班长自动选中本班所在年级）
2. **选择班级** — 选择要编辑的班级（班长自动选中本班）
3. **进入编辑表格** — 系统加载该班全部学生的分数数据

#### 分数编辑表格

| 学号 | 姓名 | 德育测评 | 学业学术 | 创新实践 | 体育基础 | 体育奖励 | 体育总分 | 美育 | 劳动 | 公益服务 | 附加分 | 总分 |
|------|------|:--------:|:--------:|:--------:|:--------:|:--------:|:--------:|:----:|:----:|:--------:|:------:|:----:|
| 20240101 | 张三 | `[85]` | `36.0` | `[10]` | 5.58 | `[2]` | 7.00 | `[4]` | `[3]` | `[8]` | `[2]` | 155.58 |

- **白色输入框** = 可编辑字段（点击直接输入数值）
- **灰色字段** = 自动计算，不可手动修改（体育基础分、体育总分、总分）

#### 实时保存机制

- 修改任一分数后，系统通过 **WebSocket 实时保存**（300ms 防抖）
- 保存成功后右上角显示 ✅ 保存状态和时间
- 保存失败时输入框标红并显示错误提示
- 支持多人同时编辑同一班级数据，修改自动同步到其他在线用户

#### 自动计算规则

- **体育总分** = 体育基础分 + 体育奖励分（修改奖励分后自动更新）
- **总分** = 德育 + 学业 + 创新 + 体育总分 + 美育 + 劳动 + 公益服务 + 附加分（任一项变化后自动重算）

#### 其他功能

- **搜索**：在搜索框输入学号或姓名快速定位学生
- **排序**：点击表头列名按该列排序
- **备注**：鼠标悬浮到单元格上点击 ✎ 图标可编辑该项的备注内容
- **返回**：点击「返回选择」可重新选择其他班级

### 4. 学生管理（管理员）

**入口**：http://localhost:3000/students

管理员可以在此页面对学生、年级、班级进行全面管理。

#### 年级/班级管理

1. **创建年级**：点击「新建年级」按钮，输入年级名称（如"2024级"）
2. **创建班级**：选择年级后，点击「新建班级」，输入班级名称（如"1班"）
3. **删除年级**：点击年级旁的删除按钮（⚠️ 将删除该年级下所有班级和学生数据）
4. **删除班级**：点击班级旁的删除按钮

#### 学生管理

1. **查看学生列表**：选择年级 → 选择班级，表格显示该班全部学生
2. **添加学生**：
   - 在表格下方输入学号和姓名，点击「添加」
3. **删除学生**：点击学生行的「删除」按钮
4. **批量删除**：
   - 按班级删除：删除整个班级的所有学生
   - 按年级删除：删除整个年级的所有学生

#### 批量导入学生

1. 点击「批量导入」按钮
2. 上传 Excel 文件（.xlsx），格式要求：
   - A列：学号
   - B列：姓名
3. 系统自动导入到当前选中的班级
4. 导入完成后显示结果：新增 N 人，跳过 M 人（已存在的学号跳过）

#### 下载模板

点击「下载模板」按钮获取标准的学生导入 Excel 模板。

### 5. 数据导入

**入口**：http://localhost:3000/import

支持三种数据批量导入。

#### 5.1 导入学业学术素质成绩（管理员）

导入学业学术素质的原始成绩文件。

1. 选择「学业学术素质」标签页
2. 可选择目标班级（留空则按学号全局匹配）
3. 上传 Excel 文件，格式要求：
   - A列：学号
   - B列：姓名
   - F列：绩点
4. 计算公式：**学业分 = (绩点 + 2.5) × 8**，四舍五入保留两位小数
5. 点击「开始导入」
6. 导入完成后显示：成功 N 条，失败 M 条

#### 5.2 导入体育基础分（管理员）

导入体育课成绩文件。

1. 选择「体育基础分」标签页
2. 可选择目标班级（留空则按学号全局匹配）
3. 上传 Excel 文件，格式要求：
   - A列：学号
   - B列：姓名
   - H列：基础分
4. 计算公式：**体育基础分 = 原始分 × 0.04**，四舍五入保留两位小数
5. 导入后体育总分自动重算

#### 5.3 导入个人综测填写表（管理员和班长）

导入个人综测填写表的汇总数据。

1. 选择「个人综测表」标签页
2. **班长**：自动限定为本班；**管理员**：可选择目标班级
3. 上传按照学校格式编制的个人综测填写表 Excel
4. 系统自动解析各维度分数和备注
5. 已有数据将被覆盖更新

#### 导入日志

页面底部「导入日志」区域显示最近的导入记录：

| 时间 | 类型 | 成功数 | 失败数 | 操作人 |
|------|------|--------|--------|--------|
| 2026-02-24 15:30 | 学业学术 | 42 | 3 | admin |

### 6. 数据导出（管理员）

**入口**：http://localhost:3000/export

提供五种导出选项：

#### 6.1 导出附件2 — 综测成绩汇总表

- 选择目标班级
- 按照学校附件2模板格式生成 Excel 文件
- 包含全部学生的各维度成绩汇总

#### 6.2 导出附件4 — 学年总评表

- 选择目标班级
- 按照学校附件4模板格式生成 Excel 文件
- 包含学生综合素质评价总评

#### 6.3 批量导出全部附件（ZIP）

- 选择目标年级
- 系统为该年级每个班级分别生成附件2和附件4
- 打包为 ZIP 文件下载
- 目录结构：`{年级}_{班级}_综测表/附件2-xxx.xlsx` + `附件4-xxx.xlsx`

#### 6.4 导出失败记录

- 导出最近导入操作中的失败记录为 Excel
- 包含列：所在行、学号、姓名、失败原因
- 用于排查导入问题

#### 6.5 导出账号列表

- 导出所有班长账号信息为 Excel
- 包含列：年级、班级、用户名、显示名称、角色

### 7. 账号管理（管理员）

**入口**：http://localhost:3000/accounts

#### 查看账号

页面分两个区域：
- **管理员账号列表**：显示用户名、显示名、创建时间
- **班长账号列表**：显示用户名、显示名、年级、班级、操作按钮

#### 批量生成班长账号

1. 点击「批量生成」按钮
2. 选择目标年级
3. 点击「确认生成」
4. 系统为该年级下每个班级自动创建一个班长账号：
   - **用户名格式**：`monitor_{年级}_{班级名}`
   - **初始密码**：随机 8 位字母数字
   - **显示名称**：`{班级名}班长`
5. 生成完成后显示创建的账号数量
6. 已有账号的班级将被跳过

#### 账号操作

- **重置密码**：点击班长账号行的「重置密码」→ 系统生成新的随机密码并显示
- **删除账号**：点击「删除」→ 确认后删除该班长账号
- **导出账号**：在导出页面可导出全部账号列表为 Excel

### 8. 系统设置

**入口**：http://localhost:3000/settings

#### 学年管理

- **创建学年**：输入学年名称（如"2025-2026学年"）点击创建

## 评分规则

系统支持以下 11 个评分维度：

| 维度 | 代码 | 分数上限 | 可编辑 | 说明 |
|------|------|:--------:|:------:|------|
| 德育测评 | `moral` | 100 | ✅ | 手动填写 |
| 学业学术素质 | `academic` | 60 | ✅ | 通常由批量导入写入 |
| 创新与实践能力 | `innovation` | 13 | ✅ | 手动填写 |
| 体育基础分 | `sports_base` | — | ❌ | 由导入写入，不可手动修改 |
| 体育奖励分 | `sports_reward` | 3 | ✅ | 手动填写 0~3 |
| 体育总分 | `sports_total` | 7 | ❌ | = 基础分 + 奖励分，自动计算 |
| 美育 | `aesthetics` | 6 | ✅ | 手动填写 |
| 劳动教育 | `labor` | 4 | ✅ | 手动填写 |
| 公益服务与社会工作 | `public_service` | 10 | ✅ | 手动填写 |
| 附加分 | `bonus` | 5 | ✅ | 手动填写 |
| 总分 | `total` | — | ❌ | 所有维度之和，自动计算 |

### 自动计算公式

```
体育总分 = 体育基础分 + 体育奖励分
总分 = 德育 + 学业 + 创新 + 体育总分 + 美育 + 劳动 + 公益服务 + 附加分
```

### 导入计算公式

```
学业学术素质 = (绩点 + 2.5) × 8        # 四舍五入保留两位小数
体育基础分 = 原始分 × 0.04              # 四舍五入保留两位小数
```

## API 接口

所有 API 以 `/api` 为前缀，需要在请求头携带 `Authorization: Bearer <token>`（登录接口除外）。

### 认证

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/login` | 登录，返回 JWT Token |
| GET | `/api/auth/me` | 获取当前用户信息 |
| PUT | `/api/auth/password` | 修改密码 |

### 用户管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/users` | 获取用户列表 |
| POST | `/api/users` | 创建用户 |
| DELETE | `/api/users/:id` | 删除用户 |
| POST | `/api/users/:id/reset-password` | 重置密码 |
| POST | `/api/users/generate-monitors/:gradeId` | 批量生成班长账号 |

### 年级 & 班级

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/grades` | 获取所有年级 |
| POST | `/api/grades` | 创建年级 |
| DELETE | `/api/grades/:id` | 删除年级 |
| GET | `/api/grades/:id/classes` | 获取班级列表 |
| POST | `/api/grades/:id/classes` | 创建班级 |
| DELETE | `/api/grades/:gradeId/classes/:classId` | 删除班级 |

### 学生管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/students` | 获取学生列表（?classId=过滤） |
| POST | `/api/students` | 添加学生 |
| PUT | `/api/students/:id` | 修改学生信息 |
| DELETE | `/api/students/:id` | 删除学生 |
| POST | `/api/students/batch/:classId` | 批量导入学生 |
| GET | `/api/students/template/download` | 下载导入模板 |

### 分数管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/scores/class/:classId` | 获取班级成绩 |
| GET | `/api/scores/student/:studentId` | 获取学生成绩 |
| PUT | `/api/scores` | 更新分数（REST 备用） |

### 数据导入

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/import/academic/:classId?` | 导入学业成绩 |
| POST | `/api/import/sports/:classId?` | 导入体育基础分 |
| POST | `/api/import/personal/:classId?` | 导入个人综测表 |
| GET | `/api/import/logs` | 获取导入日志 |

### 数据导出

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/export/attachment2/:classId` | 导出附件2 |
| GET | `/api/export/attachment4/:classId` | 导出附件4 |
| GET | `/api/export/all/:gradeId` | 批量导出（ZIP） |
| GET | `/api/export/failed-records` | 导出失败记录 |
| GET | `/api/export/accounts` | 导出账号列表 |

### 学年管理

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/academic-years` | 获取学年列表 |
| POST | `/api/academic-years` | 创建学年 |
| PUT | `/api/academic-years/:id/activate` | 激活学年 |

### WebSocket

连接地址：`ws://localhost:4000/ws?token=<JWT_TOKEN>`

| 消息类型 | 方向 | 说明 |
|---------|------|------|
| `join:class` | 客户端→服务端 | 加入班级房间 |
| `score:update` | 客户端→服务端 | 提交分数修改 |
| `score:confirmed` | 服务端→客户端 | 保存成功确认 |
| `score:updated` | 服务端→房间广播 | 分数更新通知 |
| `error` | 服务端→客户端 | 错误消息 |

```
packages/
├── package.json              # 根 package.json (npm run dev 启动全栈)
├── backend/                  # 后端
```

## 常见问题

### Q: 如何重置整个数据库？

```bash
cd packages
npm run db:reset
```

这会清除所有数据并重新创建默认管理员账号（admin/admin123）和默认学年。

### Q: 忘记了管理员密码怎么办？

运行 `npm run db:reset` 重置数据库，或直接删除 `data/comprehensive-eval.db` 文件后重新执行 `npm run db:push && npm run db:seed`。

### Q: 如何备份数据？

直接复制 `data/comprehensive-eval.db` 文件即可。SQLite 数据库是单文件存储。

### Q: 端口被占用怎么办？

- 后端端口：修改 `packages/backend/.env` 中的 `PORT` 值
- 前端端口：修改 `packages/frontend/package.json` 中 dev 脚本的 `--port` 参数
- 同时更新 `packages/frontend/astro.config.mjs` 中的代理目标地址

### Q: 导入 Excel 文件格式不对怎么办？

在「学生管理」页面点击「下载模板」获取标准模板。导入文件必须是 `.xlsx` 格式。

### Q: 多人同时编辑会冲突吗？

不会。系统通过 WebSocket 实现实时同步，多人编辑同一班级时，修改会自动广播给所有在线用户。以最后一次提交为准。

### Q: 前端开发服务器访问 API 返回 404？

确保后端已启动（端口 4000），前端通过 Vite 代理转发 `/api` 请求到后端。可用 `curl http://localhost:4000/api/health` 检查后端状态。
